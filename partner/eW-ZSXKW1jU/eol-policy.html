EOL Software Management Policy

Version: 1.0  ·  Effective: April 19, 2026

Purpose

This policy defines how MintWarden identifies and manages end-of-life (EOL) software — components for which the vendor or maintainer no longer provides security patches — to prevent exposure to unpatched vulnerabilities.

What This Covers

  • Programming language runtimes (e.g., Python, Node.js)
  • Operating system versions used in production
  • Third-party libraries and open-source dependencies
  • Infrastructure components (databases, web servers, container base images)

Detection Methods

MethodFrequency
Automated dependency scanning on all repositoriesContinuous / weekly
Language runtime lifecycle review (official release schedules)Quarterly
OS and infrastructure EOL reviewQuarterly
Manual review of dependency manifestsAt each dependency update

Response Timeline

StatusRequired ActionTimeline
EOL within 6 months (advance notice)Plan upgrade; add to engineering roadmapWithin 30 days of notice
EOL within 3 monthsBegin active upgrade workWithin 14 days of notice
Already EOL, no known CVETreat as High severity — upgrade promptlyWithin 7 days
EOL with active CVETreat as Critical — immediate responseWithin 24 hours

Exception Handling

If an immediate upgrade is not feasible, the exception must be documented with justification and compensating controls, and a hard deadline (maximum 90 days) set for resolution. Exceptions are reviewed weekly until resolved.
