Security

Access Control Policy

Version: 1.0  ·  Effective: April 19, 2026  ·  Next review: April 2027

1. Principles

  • Least privilege — Users and systems are granted the minimum access required to perform their function
  • Need to know — Access to sensitive data is limited to those with a documented business need
  • Role-based access control (RBAC) — Access is assigned by role, not individual preference
  • Default deny — Access is denied unless explicitly granted and documented

2. Access Provisioning

  • All access requests require justification and approval by the engineering lead
  • Access is assigned based on role and the least-privilege principle
  • New employee access is provisioned on or after their start date only
  • All provisioned access is recorded in the internal access inventory

3. Access Revocation

TriggerActionTimeline
Employee terminationRevoke all access across all systemsWithin 24 hours
Employee role changeModify access to match new roleWithin 48 hours
Contractor engagement endsRevoke all accessDay of departure
Suspicious activity detectedSuspend access pending investigationImmediately

4. Privileged Access

  • Production system access is restricted to essential personnel only
  • All privileged actions are logged with user identity and timestamp
  • No shared administrative accounts — every individual has a named account
  • Service accounts (used by applications) have scoped permissions and are reviewed annually

5. Periodic Access Reviews

Access is reviewed quarterly:

  • All system access is inventoried and verified against active employees and contractors
  • Any access that is excessive for the current role is removed
  • Review date and reviewer are documented

6. Consumer Access

  • Consumer accounts use email + password authentication with multi-factor authentication (MFA) required
  • Session tokens expire after a defined period of inactivity
  • Account lockout enforced after repeated failed login attempts
  • Consumers can revoke their linked bank connections at any time from account settings

7. Authentication Standards

  • Multi-factor authentication required on all internal systems storing or processing consumer data
  • Strong unique passwords required; use of a password manager is mandated for all team members
  • Secure tokens (not passwords) used for all machine-to-machine authentication
